# 自动评测与自查闭环设计 v1

## 文档目的

这份文档用于定义自动转谱系统里的“自查闭环”。

我们的目标不是只生成一版谱子，而是让系统能够：

- 自动判断当前结果和原音频差多少
- 自动识别问题主要出在哪一层
- 自动生成调试与调整建议
- 支持多轮迭代后选出更优候选谱

这份文档关注的是评测与迭代机制，不展开底层模型实现细节。

---

## 一、为什么必须做自动评测

如果没有自动评测，当前转谱流程会变成：

1. 生成一版谱
2. 人耳听一遍
3. 人工判断“不太对”
4. 手工调整参数
5. 再跑一版
6. 继续靠感觉比较

这会带来几个问题：

- 优化过程不可复用
- 很难规模化验证
- 不同人判断标准不一致
- 很难知道到底是哪一层出了问题
- 研发会持续陷入“凭感觉调参”

因此，自动评测不是附加项，而是整个系统成为产品的必要组成部分。

---

## 二、自动评测要解决什么问题

自动评测模块需要回答下面几个问题：

### 1. 当前谱子整体是否接近原音频

也就是从全局上判断：

- 节奏像不像
- 音高轮廓像不像
- 乐句结构像不像
- 听感上是否已经接近可用

### 2. 差异主要出在哪

系统不能只给一个总分，还要能定位问题，比如：

- 拍点不准
- 小节线错位
- note 太碎
- 主音抓错
- 高音异常多
- 句尾时值不稳
- 把位结果不合理

### 3. 下一轮该怎么调整

系统需要把误差转成行动建议，例如：

- 节奏误差大 -> 调整 beat/downbeat
- note 太碎 -> 提高最小音符时长阈值
- 高音异常多 -> 提高 pitch 过滤强度
- 乐句边界差 -> 重做 phrase segmentation
- 骨架不稳 -> 提高强拍音权重

### 4. 多轮结果里哪一版更好

系统需要支持：

- 保存多轮候选谱
- 记录每轮评测结果
- 最终选择综合评分最高的一版

---

## 三、自动评测在整条链路中的位置

未来完整链路应是：

1. 输入视频或混合音频
2. 提取音频并分离吉他 stem
3. 生成候选谱 v1
4. 将候选谱重新渲染成可比较的音频或事件序列
5. 与原音频进行相似度评测
6. 输出问题诊断和调整建议
7. 修改策略后生成候选谱 v2 / v3 / vN
8. 对每轮结果重复评测
9. 选择最优版本输出

因此，评测模块位于：

- 生成模块之后
- 调整模块之前

它是“生成”和“优化”之间的桥梁。

---

## 四、自动评测的核心原则

### 1. 不追求单一总分

谱子是否可用，不能只靠一个综合数值判断。

必须拆成多维度指标，否则系统会不知道到底该改什么。

### 2. 优先评测结构，再评测细节

对用户是否可用来说，优先级通常是：

1. 节奏框架
2. 乐句边界
3. 骨架音
4. 细节音
5. 技法与吉他化表达

所以评测也应遵守同样顺序。

### 3. 评测要能指导调整

每一个评测指标都应对应某种可执行的调整动作。

如果一个指标无法指导后续动作，它对闭环价值就有限。

### 4. 评测要分层

我们需要同时评测：

- 整体层
- 乐句层
- 音符层
- 可演奏性层

这样才能知道问题是“整段都偏了”，还是“只是某几句细节不对”。

---

## 五、建议的评测维度

### A. 节奏结构评测

这是最优先的评测维度。

主要看：

- BPM 偏差
- beat 对齐度
- downbeat 对齐度
- 小节边界一致性
- 音符 onset 与拍点关系是否合理
- 强拍落点是否稳定

输出指标可以包括：

- beat alignment score
- downbeat alignment score
- measure boundary score
- onset deviation mean / median

意义：

- 节奏框架错了，后面的 pitch 再准也难用

### B. 乐句结构评测

主要看：

- 句子起点是否接近
- 句尾是否接近
- 停顿位置是否接近
- 每句音符密度分布是否接近

输出指标可以包括：

- phrase boundary similarity
- pause alignment score
- phrase density similarity

意义：

- 很多谱子“听起来不像”，并不是单音错，而是句法错

### C. 骨架音评测

主要看：

- 强拍上的主音是否接近
- 长时值音是否正确
- 句尾目标音是否正确
- 主旋律轮廓是否一致

输出指标可以包括：

- backbone note match score
- phrase ending note score
- contour similarity score

意义：

- 骨架对了，整句就开始“像”了

### D. 细节音评测

主要看：

- 经过音数量是否合理
- 装饰音是否过多或过少
- 三连音、切分、短时值是否接近

输出指标可以包括：

- ornament density score
- fast note group match score
- subdivision similarity score

意义：

- 这层不应压过骨架层，但对最终听感仍然重要

### E. 音高评测

主要看：

- pitch contour 是否接近
- 关键音高是否命中
- 是否存在大量异常高音/低音
- 跳进是否不合理

输出指标可以包括：

- pitch contour similarity
- stable note pitch accuracy
- outlier pitch ratio
- excessive leap penalty

意义：

- 比起逐音百分百命中，更重要的是避免明显错误和轮廓失真

### F. 吉他可演奏性评测

主要看：

- 是否可落在合理弦位
- 把位变化是否平滑
- 是否出现明显不可能指法
- 是否符合吉他常见 phrasing

输出指标可以包括：

- fingering feasibility score
- position continuity score
- impossible fingering count

意义：

- 结果不能只“像音频”，还要“像吉他谱”

## 六、原音频与候选谱如何比较

自动评测不一定要只在符号层进行，也可以在多个表示层同时比较。

### 1. 事件层比较

比较对象：

- 原音频中检测出的事件
- 候选谱中生成的事件

比较优点：

- 结构清晰
- 方便做分层评估

适合比较：

- onset
- beat
- phrase
- note density

### 2. 轮廓层比较

比较对象：

- 原音频的 pitch contour / energy contour
- 候选谱回放后的 contour

比较优点：

- 比逐音对齐更稳
- 更接近“听起来像不像”

适合比较：

- 主旋律走势
- 句尾落点
- 节奏包络

### 3. 音频重建层比较

比较对象：

- 原音频
- 候选谱回放生成的音频

注意：

- 不要求音色完全一致
- 重点比较时间结构、旋律轮廓和重音结构

适合比较：

- onset 包络
- 节奏相似度
- melody contour

建议结论：

- 自动评测应采用“事件层 + 轮廓层 + 音频重建层”的多层比较，而不是只靠单一比对方法

---

## 七、自动诊断需要输出什么

评测系统每轮都不应只输出一个分数，而应输出结构化诊断结果。

建议输出内容包括：

- 当前版本编号
- 使用的输入 stem
- 当前主要参数
- 候选谱总体评分
- 各维度评分
- 与上一轮相比的变化
- 当前主要问题列表
- 建议调整动作列表

例如：

- 节奏评分：0.62 -> 0.74
- 骨架评分：0.58 -> 0.69
- 细节密度偏高：第 5 到 8 小节
- 高频异常值：12 个
- 建议：提高短音过滤阈值，重算 phrase segmentation

---

## 八、自动调整策略设计

评测结果必须能驱动下一轮自动调整。

建议先采用“规则驱动调整”，后面再升级为更复杂的策略选择。

### 示例映射

- beat 对齐差
  - 调整 tempo 估计策略
  - 增强 downbeat 约束
  - 尝试不同拍号候选

- note 太碎
  - 提高最小时值阈值
  - 合并相邻短 note
  - 提高 ornament 过滤强度

- 高频异常值太多
  - 限制 pitch 范围
  - 提高 pitch 平滑强度
  - 降低高频峰值权重

- 乐句边界差
  - 重做 phrase segmentation
  - 增强停顿检测
  - 增加句尾长音权重

- 可演奏性差
  - 更换指板映射策略
  - 提高把位连续性约束

---

## 九、多轮迭代机制

系统应支持同一输入下生成多轮候选结果。

建议最基础的机制如下：

1. 生成初稿 v1
2. 自动评测
3. 读取问题诊断
4. 调整参数或策略
5. 生成 v2
6. 重复若干轮
7. 记录每轮报告
8. 选出最优版本

建议每轮保存：

- 候选谱 JSON
- 导出结果
- 评测报告
- 参数快照
- 排名分数

这样研发和产品都可以复盘：

- 为什么这一轮更好
- 哪类优化最有效
- 哪类输入最容易失败

## 十、评测结果如何进入产品层

自动评测不只服务算法，也应服务用户和产品体验。

### 对用户

可以展示：

- 当前结果可信度
- 哪些小节问题较大
- 哪些地方建议重点人工检查

### 对研发

可以展示：

- 每轮算法调整效果
- 各类输入的错误分布
- 当前最主要瓶颈

### 对系统

可以驱动：

- 自动重跑
- 自动换 stem
- 自动换参数组合
- 自动选择最佳候选谱

---

## 十一、当前阶段建议先做什么

自动评测闭环不需要一步到位。

建议按下面顺序逐步落地：

### 第一步：先做基础评测报告

先加入这些最关键的分项：

- 节奏评分
- 音高轮廓评分
- note 数量合理性
- 高频异常值统计
- 短音比例统计

### 第二步：加自动问题诊断

从评测结果中自动生成：

- 主要问题列表
- 建议调整项

### 第三步：加多轮记录

支持保存：

- v1 / v2 / v3 的结果和报告
- 自动比较每轮优劣

### 第四步：加自动调参

让系统根据规则自动决定：

- 是先改节奏层
- 还是先改 pitch 层
- 是否重跑 stem 选择

---

## 十二、与整条产品链路的关系

自动评测与自查闭环不是一个附属模块，而是整个系统的中枢之一。

整条链路应变成：

1. 人工扒谱方法论
2. 自动生成谱面
3. 自动评测结果
4. 自动调整策略
5. 多轮迭代
6. 输出最优候选谱

也就是说，我们不是在做“单次转谱”，而是在做“可自我逼近原曲的转谱系统”。
