# Day 2.1 预处理方案

[English](./day2.1-preprocessing-plan.md) | [中文](./day2.1-preprocessing-plan.zh-CN.md)

目标：
- 接收本地视频或混合音频输入
- 提取可供分析使用的音频文件
- 在节奏和音高分析前先尝试做音源分离

更新后的处理链路：
1. 输入视频或混合音频
2. 提取音频
3. 统一音频格式
4. 执行音源分离
5. 选择最适合电吉他分析的 stem
6. 将选中的 stem 送入节奏和音高分析

Day 2.1 交付物：
- 一份预处理流水线定义
- 清晰的输入输出约定
- 提取音频与分离结果的目录结构
- 媒体提取与音源分离的推荐工具链
- 分离效果不佳时的回退策略

范围：
- 支持本地 `mp4`、`mov`、`wav`、`mp3`、`m4a`
- 转换为适合分析的 `wav` 音频
- 保存视频提取出的音频文件
- 保存分离后的 stem
- 允许对原始提取音频或分离后的 stem 继续分析

非范围：
- 不直接接 Bilibili 链接
- 不做云端处理
- 不做实时分离
- 不承诺所有混音都能得到干净的电吉他分离结果

推荐工具：
- `ffmpeg`：用于视频提音和音频格式统一
- `demucs`：作为第一版音源分离基线方案
- 现有 `analyzer/main.py`：继续承担节奏和音符分析

建议目录：
- `samples/raw/`：原始本地视频和混合音频
- `output/extracted/`：从视频中提取出的音频
- `output/separated/`：音源分离后的 stem
- `output/analysis/`：分析得到的 JSON 结果

实现顺序：
1. 先加媒体提取脚本
2. 再加音源分离脚本
3. 增加一个把提取、分离、分析串起来的流水线入口
4. 对比“原始提取音频”和“分离后 stem”的分析效果

成功标准：
- 至少一段本地视频能成功转成 wav
- 至少一段混合音频能产出分离 stem
- 至少一个 stem 能进入现有 Day 2 分析器
- 团队能比较哪一种输入更适合节奏初稿分析
